# 통합 GUI 앱 사용 가이드

`run_app.py` 를 실행하면 데스크톱 앱이 뜹니다. 등록·인식·삭제·검색·엑셀 추출을 한 창에서 처리합니다.

```bash
python run_app.py
```

## 화면 구성 (4개 탭)

### 🟢 실시간 인식
- **▶ 인식 시작** 을 누르면 카메라가 거울처럼 좌우 반전되어 표시됩니다.
- 같은 사람이 연속 N프레임(기본 5) 잡히면 **동작 선택 팝업**이 뜹니다.
- 팝업에는 현재 상태에서 **가능한 동작만** 버튼으로 표시됩니다(상태 머신 분기):
  - 첫 방문 → `출근`
  - 출근 후 → `외출` / `퇴근`
  - 외출 후 → `복귀`
  - 복귀 후 → `외출` / `퇴근`
  - 퇴근 후 → (동작 없음, 자동 안내)
- **버튼 클릭 또는 숫자키**로 선택합니다: `1`=출근 `2`=외출 `3`=복귀 `4`=퇴근, `ESC`=취소.
- 5초간 입력이 없으면 자동으로 닫히고 인식이 재개됩니다.

### 👥 직원 관리
- 상단 검색창에 이름/ID 를 입력하면 즉시 필터됩니다.
- **＋ 신규 등록** → 등록 다이얼로그:
  - 이름을 입력하면 **자동 ID(E001, E002…)** 가 옆에 미리 표시됩니다.
  - **📷 캠 연결**: 거울 미리보기를 보며 *현재 얼굴 캡처* 를 눌러 여러 장(권장 10장) 수집 → 3장 이상이면 등록 가능.
  - **🎞 동영상 업로드**: mp4/avi/mov/mkv 선택 → 자동으로 프레임에서 얼굴을 추출·평균.
  - 다이얼로그 안에 **등록 요령 안내문**이 표시됩니다(조명·각도 팁).
- **🗑 선택 삭제**: 표에서 행을 선택 후 삭제. 해당 직원의 근태 기록도 함께 지워집니다.
- 등록/삭제는 실시간 인식 중에도 반영됩니다(임베딩 자동 재로딩).

### 📋 근태 기록
- 이름/ID, 시작일·종료일(YYYY-MM-DD)로 검색합니다.
- **📊 엑셀 추출**: 현재 검색 결과를 `.xlsx` 로 저장합니다(헤더 스타일 포함).

### ⚙ 설정
- **DB 경로**: 찾아보기로 위치를 지정합니다. **해당 위치에 DB가 없으면 자동 생성**됩니다.
- **유사도 임계값**(기본 0.85), **투표 프레임 수**(기본 5), **GPU FP16 가속** 토글.
- **카메라**: `카메라 검색`으로 연결된 장치를 찾고, `카메라 테스트`로 선택한 카메라가 열리는지/해상도를 즉시 확인합니다. 선택한 인덱스는 인식·등록에 공통 적용됩니다.
- **설정 저장**: 설정은 실행 파일 옆 `config.json` 에 저장되어 **다음 실행 시 자동 적용**됩니다.

## DB 경로 동작 방식

- 앱은 `config.json` 의 `db_path` 를 참조합니다.
- exe 로 배포하면 `config.json` 과 (기본값일 경우) `attendance.db` 가 **exe 와 같은 폴더**에 생깁니다.
- 경로를 바꾸면 새 위치에 DB가 없을 때 자동으로 만들어 연결합니다.

## exe 로 빌드하기 (Windows)

```bash
pip install pyinstaller
pyinstaller build_exe.spec
```

- 결과물: `dist/출퇴근시스템.exe` (단일 파일, 콘솔 창 없음).
- torch/facenet 포함으로 용량이 큽니다(수백 MB~1GB+) — 정상입니다.
- **최초 실행 시** vggface2 가중치를 인터넷에서 1회 내려받습니다.
  - 오프라인 PC에 배포한다면, 인터넷 되는 PC에서 한 번 실행해 받은 캐시
    (`~/.cache/torch/checkpoints/` 또는 Windows `%USERPROFILE%\.cache\torch\`)를
    함께 복사하거나 spec 의 `datas` 에 포함하세요.

## 파일 구성

```
app/
├── config.py        # DB 경로/설정을 config.json 으로 영속화 (exe 대응)
├── core.py          # DB: 자동 ID, 등록/삭제/검색/엑셀, 상태 머신
├── face_engine.py   # MTCNN + InceptionResnetV1 (등록·인식 공용)
├── gui.py           # PyQt5 통합 UI (4개 탭 + 등록/동작 다이얼로그)
└── README_GUI.md
run_app.py           # 앱 실행 진입점
build_exe.spec       # PyInstaller exe 빌드 스펙
```

기존 CLI/학습 파이프라인(`main.py`, `train.py`, `utils/`)은 그대로 유지되며,
GUI 앱은 이를 대체하지 않고 별도 진입점으로 동작합니다.

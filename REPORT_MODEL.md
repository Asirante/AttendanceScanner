## 1. 데이터셋

### 1.1 데이터 확보
모델 비교 실험에는 공개 얼굴 데이터셋 **LFW(Labeled Faces in the Wild)** 를 사용했다.
Kaggle 배포본(`jessicali9530/lfw-dataset`)을 받아 `data/lfw-deepfunneled/{인물}/*.jpg` 구조로 배치했다.

- 사진이 **10장 이상**인 인물만 선별(`--min_images 10`) → 최종 **158명 / 4,316장**
- 인물당 사진 수는 2~80여 장으로 편차가 크다(예: George_W_Bush는 수십 장, 소수 인물은 10여 장).
  이 불균형 때문에 평가에서 정확도와 함께 Macro-F1을 병행한다(4.1 참고).

> 실제 배포 앱은 이와 별개로, 직원 얼굴을 앱에서 직접 등록한다(영상/캡처/업로드).
> 즉 LFW는 "어떤 인식 방식이 좋은가"를 검증하기 위한 실험용 데이터이며, 운영 데이터와는 분리돼 있다.

### 1.2 전처리
원본 프레임을 그대로 쓰지 않고 얼굴 영역만 정렬해 사용한다.

1. **얼굴 검출·정렬**: MTCNN으로 얼굴을 검출하고 160×160으로 정렬 크롭한다. 검출 신뢰도(`prob`)가 0.95 미만인 프레임은 버려, 흐리거나 얼굴이 작은 프레임이 학습에 섞이지 않게 한다.
2. **정규화**: 학습 시 픽셀값을 `(x - 127.5) / 128.0`으로 정규화해 InceptionResnet 입력 분포(약 -1~1)에 맞춘다. (증강을 쓰는 경우 0~1 정규화 → 증강 → -1~1 스케일 순서로 처리)

전처리 결과는 직원별 폴더에 `.pt` 텐서로 저장한다(`data/raw/{emp_id}/0001.pt …`).

### 1.3 증강 (Augmentation)
학습 데이터에만 적용하며, 검증·테스트에는 적용하지 않는다(`utils/dataset.py`).

- **RandomHorizontalFlip**: 좌우 반전으로 좌·우 얼굴 방향 변화에 강건하게.
- **ColorJitter** (brightness 0.2 / contrast 0.2 / saturation 0.1): 조명·화이트밸런스 변화에 강건하게.

증강의 효과는 실험 exp04(증강 off)와 비교해 확인한다.

### 1.4 라벨링
별도 수작업 라벨링은 필요 없다. **폴더명이 곧 직원 ID(라벨)** 이며, `utils/registry.py`의 `build_label_map`이 train 디렉토리의 직원 폴더를 정렬해 `{인덱스: 직원ID}` 매핑을 `data/label_map.json`으로 생성한다. 분류 모델의 출력 인덱스와 직원 ID가 이 매핑으로 연결된다.

### 1.5 데이터셋 구성 (train / val / test)
`utils/split_dataset.py`가 직원별로 프레임을 **70% / 15% / 15%** 로 분할한다(`random_state=42`로 재현 가능). 직원별로 프레임이 `MIN_FRAMES`(기본 10장) 미만이면 경고 후 제외해, 데이터가 너무 적은 직원이 평가를 왜곡하지 않게 한다.

| 분할 | 비율 | 용도 |
|------|------|------|
| train | 70% | 학습 |
| val | 15% | 에폭별 검증·조기 종료·최적 모델 선택 |
| test | 15% | 최종 성능 평가 (학습·검증에 미사용) |

## 2. 사용 모델

### 2.1 구조
사전학습된 **InceptionResnetV1**(facenet-pytorch, VGGFace2 가중치)을 백본으로 사용하고, 그 위에 직원 수만큼 출력하는 분류 헤드를 붙인다(`models/classifier.py`).

```
입력 얼굴 (3×160×160)
      │
 InceptionResnetV1 backbone  → 512차원 임베딩
      │
 Linear(512 → num_classes)   → 직원별 점수
```

- 백본: 얼굴을 512차원 특징 벡터로 변환 (VGGFace2 9,000명·330만 장으로 사전학습)
- 분류 헤드: 512차원 → 직원 수 만큼의 클래스 점수

### 2.2 전이학습 전략
데이터가 적어 백본을 처음부터 학습하면 과적합되므로, 사전학습 가중치를 활용하는 전이학습을 기본으로 한다.

- **동결(freeze)**: 백본을 고정하고 분류 헤드만 학습 → 적은 데이터로 빠르게 수렴.
- **부분 미세조정(fine-tune)**: 백본 마지막 일부 블록만 풀어 추가 학습(`unfreeze_last_blocks`) → 우리 데이터에 더 적응.
- **처음부터 학습(from scratch)**: 사전학습을 쓰지 않는 비교군 → 전이학습의 이점을 정량적으로 보이기 위함.

## 3. 성능 비교 방법

`run_experiments.py`로 아래 설정을 동일 데이터에서 학습·평가하고 결과를 `runs/comparison.csv`로 모은다. 비교 축은 **pretrained 사용 여부 / 미세조정 범위 / 증강 유무 / 학습률**이다.

| 실험 | pretrained | 백본 | 증강 | 학습률 | 목적 |
|------|-----------|------|------|--------|------|
| exp00 | ○ | 학습 없음 (임베딩+유사도) | - | - | 순수 VGGFace2 베이스라인 (실제 앱 방식) |
| exp01 | ○ | 전체 동결 | ○ | 1e-3 | 전이학습 기본 |
| exp02 | ○ | 마지막 2블록 미세조정 | ○ | 5e-4 | 미세조정 효과 |
| exp03 | ✕ | 처음부터 학습 | ○ | 1e-3 | 전이학습 vs 비전이 |
| exp04 | ○ | 전체 동결 | ✕ | 1e-3 | 증강 효과 |
| exp05 | ○ | 전체 동결 | ○ | 1e-4 | 학습률 영향 |
| exp06 | ○ | 임베딩 고정 + MLP 분류기 | ○ | 1e-3 | 직접 설계한 분류기(은닉 2층, BatchNorm·Dropout) |
| exp07 | ○ | 임베딩 고정 + ArcFace | ○ | 1e-3 | 각도 마진 손실. margin·scale 12조합 그리드 탐색 |

- exp00은 학습하지 않는다. train 이미지로 사람별 평균 임베딩(프로토타입)을 만들고, test 임베딩을 가장 가까운(코사인 유사도) 프로토타입으로 분류한다. 이는 실제 출퇴근 앱이 쓰는 방식과 동일하며, "사전학습 모델을 그대로 쓰면 어느 정도 나오는가"의 기준선이 된다.
- exp06은 고정된 임베딩 위에 직접 설계한 MLP(512→512→256→클래스, BatchNorm + Dropout 0.4)를 학습한다. 백본을 다시 통과시키지 않고 미리 추출한 임베딩으로 학습하므로 매우 빠르며, "기성 임베딩을 그대로 쓰지 않고 그 위에 분류기를 직접 설계·학습"한 경우의 성능을 본다.
- exp07은 얼굴 인식의 대표적 기법인 **ArcFace(각도 마진 손실)** 를 적용한다. 일반 softmax는 정답 점수를 높이기만 하지만, ArcFace는 임베딩과 클래스 중심의 각도 θ에 대해 정답 클래스에만 **cos(θ + m)** 의 마진을 부여하여, 같은 사람은 더 조밀하게·다른 사람과는 더 멀게 임베딩 공간을 재정렬한다. 마진 `m`과 스케일 `s`가 핵심 하이퍼파라미터이며, 본 실험에서는 **margin 4종(0.2/0.35/0.5/0.7) × scale 3종(16/32/64) = 12조합**을 그리드로 탐색했다.
- 공통: 옵티마이저 Adam, weight_decay 1e-4, CosineAnnealing 스케줄러, val 정확도 기준 조기 종료, val 최고 시점 가중치 저장.

## 4. 성능 및 측정 기준

### 4.1 측정 지표
- **정확도(Accuracy)**: 전체 맞춘 비율.
- **Macro-F1**: 직원별 F1의 평균 — 특정 인원에 데이터가 쏠려도 균형 있게 평가.
- **혼동 행렬(Confusion Matrix)**: 어떤 직원이 누구와 혼동되는지 확인.
- **추론 시간(Inference Time) / FPS**: 실시간 출퇴근용이므로 속도도 평가. 실시간 처리 기준으로 **FPS ≥ 15** 를 목표로 한다. (`evaluate.py`가 측정)

### 4.2 결과

| 실험 | 방식 | test 정확도 | macro-F1 | FPS |
|------|------|------------|----------|-----|
| exp00 | 유사도(학습 없음) | 0.9601 | 0.9601 | 89.2 |
| exp01 | 분류 — 동결 | 0.9668 | 0.9567 | 42.2 |
| exp02 | 분류 — 미세조정 | 0.9640 | 0.9542 | 42.7 |
| exp03 | 분류 — 처음부터 학습 | 0.7441 | 0.6250 | 69.2 |
| exp04 | 분류 — 동결 + 증강끔 | 0.9682 | 0.9607 | 93.6 |
| exp05 | 분류 — 낮은 학습률 | 0.7870 | 0.6222 | 82.7 |
| exp06 | 임베딩 + MLP 분류기 | 0.9696 | 0.9624 | 3472.7 |
| exp07 | 임베딩 + ArcFace (최적 조합) | **0.9723** | 0.9632 | 7290.7 |

- ArcFace 최적 조합(margin 0.5, scale 16)이 **test 0.9723으로 전체 최고**를 기록했다.
  직접 설계한 MLP(exp06, 0.9696)와 단순 유사도(exp00, 0.9601)가 그 뒤를 잇는다.
- 다만 사전학습 기반 방식들(exp00~02, exp04, exp06, exp07)은 모두 0.96~0.97 구간에 몰려 있어
  **서로의 차이는 1%p 안팎으로 유의미하다고 보기 어렵다**(test 표본 723장).
  최고(exp07 0.9723)와 학습 없는 임베딩(exp00 0.9601)의 차이도 1.2%p에 그친다.
- 백본을 처음부터 학습한 exp03(0.74)과 학습이 덜 수렴한 exp05(0.79)만 뚜렷이 낮았다.

> exp06·07의 FPS(수천)는 미리 추출한 임베딩에서 분류 헤드만 통과하는 속도라 다른 실험과 직접 비교 대상은 아니다.
> 실제 인식에서는 얼굴 검출·임베딩 추출(MTCNN + 백본) 시간이 더해지며, 이는 exp00~05와 동일하다.

### 4.3 ArcFace 하이퍼파라미터 그리드 (exp07)

margin × scale 12조합을 모두 학습·평가한 결과(`runs/arcface_grid.csv`):

| margin \ scale | 16 | 32 | 64 |
|----------------|-----|-----|-----|
| **0.2**  | 0.9710 | 0.9696 | 0.9668 |
| **0.35** | 0.9696 | 0.9710 | 0.9710 |
| **0.5**  | **0.9723** | 0.9710 | 0.9710 |
| **0.7**  | 0.9710 | 0.9710 | 0.9710 |

(값은 test 정확도)

- 12조합 전부 0.9668~0.9723에 분포해 **편차가 0.55%p로 매우 작다.** 즉 이 데이터 규모에서는
  마진·스케일 선택에 성능이 크게 좌우되지 않을 만큼 임베딩이 이미 안정적이다.
- 그래도 경향은 보인다: **margin 0.5 부근이 전반적으로 가장 안정적**이며(ArcFace 논문 권장값과 일치),
  margin이 너무 작고(0.2) scale이 너무 큰(64) 극단 조합에서 소폭 하락했다.
- ArcFace가 12조합 중 일부에서 다른 방식보다 근소하게 높았으나, 그 차이는 통계적으로 유의하지 않다.

### 4.4 임베딩 분석 (검증·시각화)

학습 외에, 임베딩 자체의 품질을 정량·시각적으로 분석했다(`analyze_embeddings.py`).

- **t-SNE 시각화** (`runs/figs/tsne.png`): 512차원 임베딩을 2차원으로 투영하면
  같은 사람의 얼굴이 뚜렷한 군집을 이룬다. 사전학습 임베딩이 인물을 잘 분리함을 시각적으로 보여준다.
- **혼동 행렬 히트맵** (`runs/figs/confusion.png`): 오분류는 주로 표본이 적은 소수 인물에서 발생했다.
- **검증 ROC / EER** (`runs/figs/roc.png`): 모든 얼굴 쌍의 코사인 유사도로 "같은 사람인지"를 판별하는
  검증 관점의 성능을 측정했다. **AUC 0.9727, EER 5.77%** 로, 임베딩만으로도 신뢰할 만한 분리력을 보인다.
  EER 지점의 유사도 임계값을 데이터로부터 도출할 수 있어, 실제 시스템의 인식 임계값 설정 근거로 활용할 수 있다.

> 참고: 이 ROC의 임계값은 "모든 쌍 비교(verification)" 기준이며, 실제 앱의 1:N 매칭 임계값과는
> 측정 방식이 다르다. 분석의 의의는 임계값을 임의로 정하지 않고 데이터 기반으로 정당화할 수 있다는 점이다.

> 결론은 4.5에서 정리한다.

### 4.5 결론

- **사전학습(전이학습)이 결정적이다.** 백본 없이 처음부터 학습한 exp03(0.74)과 수렴이 덜 된 exp05(0.79)는
  크게 낮았다. 4,316장 규모로는 백본을 처음부터 학습하기에 데이터가 부족하다.
- **점진적으로 난이도를 올려가며 성능 상한을 확인했다.** 단순 유사도(exp00, 0.9601) → 분류(exp01·02·04) →
  직접 설계한 MLP(exp06, 0.9696) → 얼굴 인식 SOTA 기법인 ArcFace 12조합 그리드(exp07, 최고 0.9723)까지
  올려봤다. 성능은 0.96에서 0.97로 미미하게 상승했을 뿐이며, 사전학습 기반 방식들 간 차이는 1%p 안팎으로
  통계적으로 유의하지 않다(test 723장).
- **그럼에도 실제 앱은 유사도(exp00) 방식을 채택한다.** 분류·MLP·ArcFace 방식은 출력 클래스가 학습 시점의
  인원으로 고정되어, **새 직원이 들어올 때마다 재학습**이 필요하다. 반면 유사도 방식은 사전학습 임베딩만으로
  동작해 **재학습 없이 등록만으로 직원을 추가**할 수 있다. 출퇴근처럼 인원이 수시로 바뀌는 환경에서는
  운영 편의성이 결정적이며, 정확도 차이가 1%p 안팎으로 미미하므로 유사도 방식이 실용적으로 가장 적합하다.
- 정리하면 **ArcFace까지 동원해 성능 상한(exp07)을 확인하고, 운영상 최적(exp00)과의 트레이드오프를 근거로
  운영 환경에 맞는 방식을 선택**했다. 성능 차이가 미미한 반면 재학습 비용 차이는 크기 때문이다.

## 5. 학습 로그
학습 과정은 TensorBoard로 기록한다(`train.py`의 `SummaryWriter`, `runs/{exp_name}/`).

- `Loss/train`: 에폭별 학습 손실
- `Acc/val`: 에폭별 검증 정확도
- `LR`: 학습률 변화(CosineAnnealing)

확인: `tensorboard --logdir runs` 실행 후 브라우저에서 곡선을 캡처해 보고서에 첨부한다.

## 7. 실행 순서 (재현 방법)

학습 실험은 공개 데이터셋 **LFW(Labeled Faces in the Wild)** 를 사용한다. (연구용 공개 데이터셋)

```bash
# 1) LFW 데이터셋을 Kaggle 에서 받아 data/ 에 배치
#    https://www.kaggle.com/datasets/jessicali9530/lfw-dataset
#    압축을 풀면 data/lfw-deepfunneled/{인물}/*.jpg 구조가 된다.
#    (함께 들어있는 csv 메타파일은 사용하지 않으므로 무시해도 됨)

# 2) MTCNN 정렬 → data/raw/{인물}/*.pt
#    사진이 10장 미만인 인물은 자동 제외 (--min_images 로 조정 가능)
python utils/prepare_lfw.py --lfw_dir data/lfw-deepfunneled --min_images 10

# 3) train/val/test 분할 (70:15:15)
python utils/split_dataset.py

# 4) 비교 실험(exp00~05) 일괄 실행 → runs/comparison.csv
python run_experiments.py

# 5) (추가) 임베딩 위 MLP 분류기 학습 → exp06
python train_mlp.py

# 6) (추가) ArcFace margin/scale 12조합 그리드 → exp07, runs/arcface_grid.csv
python train_arcface.py

# 7) (추가) 임베딩 분석 그림 생성 → runs/figs/ (t-SNE, ROC, 혼동행렬)
python analyze_embeddings.py

# 8) 학습 곡선 확인
tensorboard --logdir runs
```

> min_images 10 기준으로 선별하면 최종 **158명 / 4,316장**(test 723장)이 된다. 값을 낮추면
> 클래스 수는 늘지만 인물당 표본이 줄어 정확도 해석에 주의해야 한다.
>
> 동봉된 `utils/download_lfw.py` 는 원 배포처(umass.edu) 도메인이 응답하지 않는 환경에서는
> 동작하지 않을 수 있다. 그럴 때는 위와 같이 Kaggle 등에서 직접 받아 배치한다.

> 실제 운영 시스템은 이와 별개다. 직원 얼굴을 앱의 등록 기능(캡처/영상으로 등록)으로 직접 등록하며, 위 학습 과정 없이 사전학습 임베딩만으로 동작한다.

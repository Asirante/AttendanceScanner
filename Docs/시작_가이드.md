# 시작 가이드 — AI 안면인식 출퇴근 시스템

압축을 푼 직후부터 실시간 인식이 돌아갈 때까지, 순서대로 따라 하면 됩니다.
**위에서 아래로 한 단계씩** 진행하세요. 건너뛰면 뒤에서 막힙니다.

> 💡 **이 시스템이 하는 일**: 웹캠으로 얼굴을 인식해서 직원을 알아보고, 출근·외출·복귀·퇴근을 SQLite DB에 기록합니다. 신규 직원은 **재학습 없이** 등록 한 번이면 됩니다.

---

## 0. 시작 전 준비물 체크

진행 전에 아래가 갖춰져 있는지 확인하세요.

| 항목 | 필요 사항 | 확인 방법 |
|------|-----------|-----------|
| Python | 3.9 ~ 3.11 | `python --version` |
| GPU(선택) | NVIDIA + CUDA 11.8 | 없으면 CPU로도 동작(느림) |
| 웹캠 | USB 또는 내장 카메라 | 직원 등록·인식에 필수 |
| NAS(선택) | DB 저장 경로 | 없으면 로컬 폴더로 대체 가능 |

> GPU가 없어도 일단 동작은 합니다. 다만 실시간 FPS가 낮아질 수 있습니다.

## 1. 압축 풀기 & 폴더 안으로 이동

압축을 이미 덮어쓰셨다니, 압축을 푼 위치에서 `project` 폴더로 들어갑니다.

```bash
# 압축을 푼 폴더로 이동 (예시 경로 — 본인 경로로 바꾸세요)
cd ~/Downloads/project

# 폴더 안에 이게 보이면 제대로 들어온 겁니다:
#   run_app.py  train.py  requirements.txt  app/  utils/  tests/ ...
ls
```

`run_app.py`, `requirements.txt`, `app/` 폴더가 보이면 다음 단계로 갑니다.
안 보이면 한 단계 더 들어가야 할 수 있습니다 (`cd project`).

---

## 2. 가상환경 만들기 (권장)

시스템 Python을 더럽히지 않도록 프로젝트 전용 환경을 만듭니다.

```bash
# 가상환경 생성
python -m venv .venv

# 활성화
source .venv/bin/activate          # macOS / Linux
# Windows(PowerShell):  .venv\Scripts\Activate.ps1
# Windows(cmd):         .venv\Scripts\activate.bat
```

활성화되면 터미널 줄 맨 앞에 `(.venv)` 가 붙습니다. **이게 안 보이면 활성화가 안 된 것**이니 위 명령을 다시 확인하세요.

---

## 3. 패키지 설치

```bash
pip install -r requirements.txt
```

`requirements.txt` 첫 줄에 PyTorch CUDA 인덱스가 들어 있어서, GPU용 PyTorch가 자동으로 설치됩니다. 설치는 처음 한 번만 몇 분 걸립니다.

설치가 끝나면 PyTorch와 CUDA가 잡혔는지 확인합니다.

```bash
python -c "import torch; print(torch.cuda.is_available(), torch.__version__)"
```

- `True 2.2.0` → GPU 정상 인식 (이상적)
- `False 2.2.0` → GPU는 안 잡혔지만 CPU로 동작 (괜찮음, 느릴 뿐)

---

## 4. 환경 변수(.env) 설정

DB 파일을 어디에 저장할지 알려주는 단계입니다. 경로를 코드에 직접 박지 않고 `.env` 파일로 관리합니다.

```bash
# 예시 파일을 복사해서 내 설정 파일 만들기
cp .env.example .env
```

그다음 `.env` 파일을 열어 경로를 환경에 맞게 고칩니다.

```bash
# .env 파일 내용
NAS_DB_PATH=/mnt/nas/attendance.db
```

**상황별 권장 값**

- **NAS를 쓰는 경우**: NAS가 마운트된 실제 경로로 지정 (예: `/mnt/nas/attendance.db`)
- **NAS 없이 로컬에서 테스트**: 프로젝트 안 경로로 지정

```bash
# 로컬 테스트 예시 — 이렇게 바꾸면 NAS 없이도 됩니다
NAS_DB_PATH=./attendance.db
```

> ⚠️ GUI **설정** 탭에서 DB 경로를 지정하면 됩니다. NAS 없이 테스트하려면 앱 폴더 안 경로(예: `attendance.db`)로 두세요.

## 5. DB 만들기 (자동/수동)

DB는 **따로 만들 필요 없이** 코드가 처음 실행될 때 자동으로 생성됩니다.
그래도 미리 만들어두고 싶다면 아래 한 줄을 실행하세요.

```bash
# (선택) DB를 미리 생성
python -c "from utils.database import AttendanceDB; db=AttendanceDB(); print('DB 생성 완료'); db.close()"
```

`DB 생성 완료` 가 출력되면 `Employees` / `AttendanceLog` 테이블이 만들어진 것입니다.

---

## 6. 잘 돌아가는지 먼저 테스트 (중요)

웹캠을 켜기 전에, 핵심 로직(출퇴근 상태 전이)이 정상인지 자동 테스트로 확인합니다.
이건 카메라·GPU 없이도 됩니다.

```bash
python -m pytest tests/ -v
```

pytest가 없다는 오류가 나오면 둘 중 하나로 하세요.

```bash
pip install pytest          # 설치 후 위 명령 재실행
# 또는 설치 없이 바로 실행:
python tests/test_state_machine.py
```

**`6 passed` 또는 `OK` 가 나오면 통과**입니다. 다음으로 넘어가세요.

## 7. 통합 GUI 앱 실행 ⭐ (여기부터 GUI)

7번부터는 명령어가 아니라 **데스크톱 앱**으로 진행합니다. 앱 하나에서 등록·인식·삭제·검색·엑셀 추출을 모두 합니다.

```bash
python run_app.py
```

창이 뜨면 위쪽에 4개 탭이 보입니다: **🟢 실시간 인식 / 👥 직원 관리 / 📋 근태 기록 / ⚙ 설정**.

### 7-1. 먼저 DB 경로 설정 (⚙ 설정 탭)

1. **⚙ 설정** 탭으로 이동합니다.
2. **DB 경로** 에서 *찾아보기* 를 눌러 DB를 둘 위치를 지정합니다.
   - NAS를 쓰면 NAS 경로, 로컬 테스트면 앱 폴더 안 아무 경로면 됩니다.
   - **그 위치에 DB가 없으면 자동으로 생성**되니 미리 만들 필요 없습니다.
3. **💾 설정 저장** 을 누릅니다. 이 설정은 실행 파일 옆 `config.json` 에 저장되어 다음 실행 때 자동 적용됩니다.

### 7-2. 직원 등록 (👥 직원 관리 탭)

1. **👥 직원 관리** 탭 → **＋ 신규 등록** 클릭.
2. **이름**을 입력하면 옆에 **자동 ID(E001, E002…)** 가 미리 표시됩니다. (ID는 직접 입력 안 함)
3. 등록 방법을 선택합니다.
   - **📷 캠 연결**: 거울처럼 화면이 보입니다. 얼굴을 맞추고 **‘현재 얼굴 캡처’** 를 눌러 여러 장(권장 10장) 모읍니다. 고개를 아주 조금씩 좌우·상하로 돌리며 캡처하면 인식이 더 잘 됩니다. **3장 이상**이면 등록 버튼이 활성화됩니다.
   - **🎞 동영상 업로드**: 5~10초짜리 얼굴 영상을 고르면 자동으로 처리합니다.
4. **✅ 등록 완료** 클릭 → 끝. 더 추가할 직원이 있으면 2번부터 반복합니다.

> 다이얼로그 안에 등록 요령 안내문(조명·각도 팁)이 항상 표시됩니다.
> ✅ **핵심**: 신규 직원을 추가해도 모델을 다시 학습할 필요가 없습니다. 등록만 하면 바로 인식됩니다.

---

## 8. 실시간 인식 (🟢 실시간 인식 탭)

1. **🟢 실시간 인식** 탭 → **▶ 인식 시작** 클릭. 카메라가 거울처럼 표시됩니다.
2. 등록된 직원이 화면에 들어오면, 연속 5프레임 일치 시 **동작 선택 팝업**이 뜹니다.
3. 팝업에는 **지금 가능한 동작만** 버튼으로 나옵니다(상태에 따라 자동 분기). 버튼 클릭 또는 숫자키로 선택합니다.

| 키 | 동작 |
|----|------|
| `1` | 출근 |
| `2` | 외출 |
| `3` | 복귀 |
| `4` | 퇴근 |
| `ESC` | 취소 |

> 5초간 입력이 없으면 팝업이 자동으로 닫히고 인식이 재개됩니다. (프로그램 종료 버튼은 팝업에 없습니다 — 종료는 창을 닫으면 됩니다.)

**상태 전이 규칙** (이 분기대로만 버튼이 표시됩니다)

```
(오늘 첫 방문) → 출근
출근           → 외출 또는 퇴근
외출           → 복귀
복귀           → 외출 또는 퇴근
퇴근           → (당일 종료, 동작 없음)
```

1. **📋 근태 기록** 탭에서 이름/ID, 시작일·종료일(YYYY-MM-DD)로 검색합니다.
2. **📊 엑셀 추출** 을 누르면 현재 검색 결과가 `.xlsx` 파일로 저장됩니다(헤더 스타일 포함).

# (1) 학습 데이터 수집 — 직원별 동영상에서 프레임 추출

## 자주 막히는 곳 (문제 해결)

| 증상 | 원인 | 해결 |
|------|------|------|
| `(.venv)` 가 안 보임 | 가상환경 미활성화 | 2번 활성화 명령 재실행 |
| `ModuleNotFoundError: No module named 'utils'` | 프로젝트 폴더 밖에서 실행 | `project/` 폴더 안에서 실행 |
| `ModuleNotFoundError: No module named 'torch'` | 패키지 미설치 | 3번 `pip install -r requirements.txt` |
| `NAS 마운트 안 됨` 경고 | NAS 경로가 안 잡힘 | `.env`를 `./attendance.db`로 변경(4번) |
| 카메라 창이 안 열림 | 다른 앱이 웹캠 점유 / 권한 | 카메라 쓰는 앱 종료, OS 카메라 권한 허용 |
| `임베딩 수집 실패` | 얼굴이 안 잡힘 | 밝은 곳에서 정면, 카메라 가까이 |
| 자꾸 `unknown` 으로 뜸 | 유사도 임계값이 높음 | GUI 설정 탭에서 유사도 임계값 0.70 → 0.65로 낮춤 |
| 다른 사람으로 오인식 | 임계값이 낮음 | `SIM_THR`를 `0.75`로 높임 |
| FPS가 너무 낮음 | CPU 추론 | GPU 사용 권장 / 설정 탭에서 FP16 가속 켜기 |
| 앱이 안 켜짐 (`No module named PyQt5`) | GUI 패키지 미설치 | `pip install -r requirements.txt` 재실행 |
| 등록 다이얼로그 캠이 검음 | 카메라 권한/점유 | 다른 앱 종료, OS 카메라 권한 허용 |
| exe 첫 실행이 느림 | vggface2 가중치 1회 다운로드 | 인터넷 연결 상태로 한 번 실행(이후 캐시됨) |

## 한눈에 보는 전체 순서

```
1. cd project           ← 폴더 진입
2. python -m venv .venv && source .venv/bin/activate   ← 가상환경
3. pip install -r requirements.txt                     ← 패키지(PyQt5 포함)
4. cp .env.example .env  → 경로 수정(선택, GUI에서도 설정 가능)
5. (자동) DB 생성
6. python -m pytest tests/ -v                          ← 동작 테스트
7. python run_app.py                                    ← GUI 앱 실행 ▶
     ⚙ 설정 탭에서 DB 경로 저장
     👥 직원 관리 → ＋신규 등록 (캠/동영상 + 이름)
8. 🟢 실시간 인식 탭 → ▶ 인식 시작
9. 📋 근태 기록 탭 → 검색 + 엑셀 추출
```

> 💡 **exe로 배포**하려면: `pip install pyinstaller` 후 `pyinstaller build_exe.spec` →
> `dist/출퇴근시스템.exe` 가 생깁니다. 자세한 내용은 `app/README_GUI.md` 참고.

여기까지 따라오면 출퇴근 인식이 동작합니다. 막히는 부분은 위 "문제 해결" 표를 먼저 확인하세요.

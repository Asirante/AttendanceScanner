AI 기반 실시간 안면인식 출퇴근 시스템

PyTorch Metric Learning 및 벡터 유사도 매칭을 활용한 Zero-Retraining 근태 관리 솔루션

1. 문제 정의 및 기대 효과

🎯프로젝트 목표

약 30명 규모의 중소 사업장을 위한 실시간 안면인식 출퇴근 시스템 구축. 기존 QR코드 태그 방식의 번거로움과 대리 출결 문제를 해결하고, 95% 이상의 인식 정확도를 달성함.

🚀기대 효과 (Scalability)

  • Agile Development 소규모 파일럿 테스트 지원: 3~4명 규모로 초기 모델 검증 및 시스템 로직(상태머신) 테스트를 완벽히 수행 가능.
  • Zero-Retraining 유연한 확장성: 파일럿 테스트 완료 후, 모델의 코드 수정이나 재학습 없이 단순 DB 등록만으로 즉시 30명(혹은 그 이상) 규모의 실무 환경에 배포 가능.

2. 데이터 셋 설명

동영상 기반 얼굴 프레임 데이터 (직원 30명)

직원당 약 5~10초의 얼굴 회전 동영상을 촬영 후, 아래 파이프라인을 거쳐 데이터를 구축합니다.

등록 (Enrollment)

동영상에서 샘플링된 50여 장의 프레임으로 기준이 되는 벡터(Prototype)를 생성하여 DB에 저장.

임계값 최적화 (Validation)

등록자 본인/타인의 영상 교차 검증을 통해 최적의 거리 임계값(Threshold) 산출.

테스트 (Test)

학습/등록에 관여하지 않은 새로운 환경(조명/각도)의 데이터를 통한 최종 성능 평가.

데이터 전처리: Frame Skipping (프레임 샘플링) MTCNN Confidence Filtering (흔들림 제거)

4. 성능 비교 방법 (실험 계획 세분화)

🧪 4-1. 텐서 거리(유사도) 측정 방식 비교

  • Cosine Similarity: 벡터의 크기(조명 밝기 등)에 무관하게 방향성(얼굴 특징)만으로 유사도를 판별하는 PyTorch F.cosine_similarity() 연산 평가.
  • Euclidean Distance (L2 Norm): 512차원 공간 상의 물리적 거리를 계산하여 비교. Cosine 방식 대비 조명 변화 강건성(Robustness) 분석.

📊 4-2. 기준 벡터(Prototype) 텐서 생성 전략

  • 단일 이미지 저장: 각 직원당 사진 1장의 512차원 텐서만을 저장할 때의 인식률.
  • 다중 프레임 평균 (Mean Tensor): 동영상에서 추출한 50개 프레임의 텐서를 PyTorch의 torch.mean() 연산을 통해 하나의 정교한 기준 텐서로 압축하여 등록할 때의 성능 향상률 비교.

⚙️ 4-3. 최적 임계값(Threshold) 도출 분석

  • FAR vs FRR Trade-off: 타인을 본인으로 오인할 확률(FAR)과 본인을 인식하지 못할 확률(FRR)이 교차하는 지점(EER)을 분석하여 최적의 Cosine Similarity Threshold(예: 0.85) 도출 실험.

5. 성능 측정 기준

  • Accuracy (정확도): 최종 목표 95% 이상. (파일럿 인원 및 최종 30명 대상 교차 검증)
  • False Acceptance Rate (오인식률): 보안과 직결되므로 등록되지 않은 외부인 사진을 0% 확률로 통과시키는지 여부(Threshold 검증).
  • Inference Time (추론 시간): 모델의 텐서 추출 및 SQLite 순회 속도를 측정하여 1 프레임당 인식 속도(목표치: 100ms 이내) 검증.

6. UI 프로세스 및 출퇴근 상태 머신 로직

📸 OpenCV 인터페이스

화면 중앙에 사람 테두리(실루엣) 오버레이를 띄웁니다. 사용자가 실루엣에 얼굴을 맞추면 Bounding Box 좌표를 검사하여 일정 영역 내에 들어왔을 때만 안면 인식을 트리거(Trigger)하여 연산 낭비를 막습니다.

🗄️ NAS 연동 SQLite 상태 머신 및 스키마

직원 신규 등록 시 모델 재학습 없이 Employees 테이블에 임베딩 벡터(BLOB)만 추가하며, 인식 시 DB와 유사도 연산을 진행합니다.

📌 SQLite 간이 스키마 (Schema)

CREATE TABLE Employees (
    emp_id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    embedding BLOB NOT NULL -- PyTorch Tensor의 Numpy bytes 저장
);

CREATE TABLE AttendanceLog (
    log_id INTEGER PRIMARY KEY AUTOINCREMENT,
    emp_id TEXT,
    date TEXT,       -- YYYY-MM-DD
    time TEXT,       -- HH:MM:SS
    state TEXT       -- '출근', '퇴근', '외출', '복귀'
);
// 당일(date) AttendanceLog 조회 결과에 따른 UI 분기
